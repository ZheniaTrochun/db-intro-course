# Довідник ключових термінів курсу

## Загальні поняття (Лекція 1)

- База даних (Database, DB) - організований набір даних, що моделює певний процес або аспект реального світу
- СУБД (Database Management System, DBMS) - програмне забезпечення для створення, зберігання та управління базами даних (PostgreSQL, MySQL, MongoDB)
- SQL (Structured Query Language) - стандартизована мова для роботи з реляційними базами даних
- DDL (Data Definition Language) - частина SQL для визначення структури БД (CREATE, ALTER, DROP)
- DML (Data Manipulation Language) - частина SQL для роботи з даними (SELECT, INSERT, UPDATE, DELETE)
- DCL (Data Control Language) - частина SQL для управління правами доступу (GRANT, REVOKE)
- TCL (Transaction Control Language) - частина SQL для управління транзакціями (BEGIN, COMMIT, ROLLBACK)
- CRUD (Create, Read, Update, Delete) - чотири базові операції з даними: INSERT, SELECT, UPDATE, DELETE
- OLTP (Online Transaction Processing) - робоче навантаження з великою кількістю коротких транзакцій (типове для веб-застосунків)
- OLAP (Online Analytical Processing) - робоче навантаження з складними аналітичними запитами на великих об'ємах даних

---

## ER-діаграми (Лекція 2)

- ER-діаграма (Entity-Relationship Diagram) - візуальне представлення структури бази даних: сутностей, їх атрибутів та зв'язків між ними
- Сутність (Entity) - об'єкт реального світу, про який зберігається інформація (Студент, Курс, Викладач)
- Атрибут (Attribute) - властивість або характеристика сутності (ім'я, email, дата народження)
- Зв'язок (Relationship) - асоціація між двома або більше сутностями (Студент записується на Курс)
- Кардинальність (Cardinality) - кількість екземплярів однієї сутності, що може бути пов'язана з екземплярами іншої (1:1, 1:N, M:N)
- Зв'язок "один до одного" (One-to-One, 1:1) - кожен екземпляр A пов'язаний рівно з одним екземпляром B, і навпаки
- Зв'язок "один до багатьох" (One-to-Many, 1:N) - один екземпляр A може бути пов'язаний з багатьма екземплярами B, але кожен B - лише з одним A
- Зв'язок "багато до багатьох" (Many-to-Many, M:N / M:M) - кожен екземпляр A може бути пов'язаний з багатьма B, і кожен B - з багатьма A
- Первинний ключ (Primary Key, PK) - атрибут або набір атрибутів, що унікально ідентифікує кожен запис у таблиці
- Похідний атрибут (Derived Attribute) - атрибут, значення якого обчислюється з інших атрибутів (наприклад, розмір групи студентів)

## Таблиці та DDL (Лекція 3)

- Таблиця (Table / Relation) - структура зберігання даних у реляційній БД, складається з рядків і колонок
- Рядок (Row / Record / Tuple) - один запис у таблиці, що містить набір значень для кожної колонки
- Колонка (Column / Field / Attribute) - визначає тип даних, що зберігаються в конкретній позиції кожного рядка
- Схема (Schema) - логічна структура бази даних: набір таблиць, їх колонок, типів даних та обмежень
- Тип даних (Data Type) - визначення допустимих значень колонки (INTEGER, VARCHAR, BOOLEAN, DATE, JSONB і тд)
- Обмеження (Constraint) - правило, що гарантує цілісність даних у таблиці
- Первинний ключ (Primary Key) - колонка або набір колонок, що унікально ідентифікує рядок (NOT NULL + UNIQUE)
- Зовнішній ключ (Foreign Key, FK) - колонка, що посилається на первинний ключ іншої таблиці, забезпечуючи зв'язок між таблицями
- NOT NULL - обмеження, що забороняє порожні значення в колонці
- UNIQUE - обмеження, що забороняє дублювання значень у колонці
- CHECK - обмеження, що перевіряє значення за заданою умовою
- DEFAULT - значення за замовчуванням, що автоматично підставляється при вставці, якщо явно не вказано значення колонки 
- Каскадне видалення (CASCADE DELETE) - автоматичне видалення залежних рядків при видаленні батьківського рядка

---

## DML та CRUD (Лекція 4)

- INSERT - додавання нових рядків до таблиці
- SELECT - вибірка даних з таблиці
- UPDATE - зміна існуючих даних у таблиці
- DELETE - видалення рядків з таблиці
- TRUNCATE - швидке видалення всіх рядків з таблиці (скидає лічильники автоінкременту)
- WHERE - фільтрація рядків за умовою
- ORDER BY - сортування результатів запиту
- DISTINCT - усунення дублікатів з результатів запиту
- Псевдонім (Alias, AS) - тимчасове ім'я для колонки або таблиці в запиті
- NULL - спеціальне значення, що означає відсутність даних (перевіряється через IS NULL / IS NOT NULL)
- М'яке видалення (Soft Delete) - позначення запису як видаленого (наприклад, через прапорець is_deleted) замість фізичного видалення

---

## JOIN та операції над множинами (Лекція 5)

- JOIN - об'єднання рядків з двох або більше таблиць на основі спільних значень
- INNER JOIN - повертає лише рядки, що мають збіги в обох таблицях
- LEFT JOIN (LEFT OUTER JOIN) - повертає всі рядки з лівої таблиці + збіги з правої (NULL де збігів немає)
- RIGHT JOIN (RIGHT OUTER JOIN) - повертає всі рядки з правої таблиці + збіги з лівої
- FULL OUTER JOIN - повертає всі рядки з обох таблиць, NULL де збігів немає
- CROSS JOIN - декартів добуток - кожен рядок з першої таблиці з'єднується з кожним рядком другої
- Anti-Join - паттерн для знаходження рядків БЕЗ збігів (LEFT JOIN + WHERE ... IS NULL)
- UNION - об'єднання результатів двох запитів (без дублікатів)
- UNION ALL - об'єднання результатів двох запитів (з дублікатами)
- INTERSECT - повертає лише спільні рядки двох запитів
- EXCEPT - повертає рядки з першого запиту, яких немає в другому

---

## Агрегація та віконні функції (Лекція 6)

- Агрегатна функція (Aggregate Function) - функція, що обчислює одне значення з набору рядків (COUNT, SUM, AVG, MIN, MAX)
- GROUP BY - групування рядків за значеннями однієї або кількох колонок для агрегації
- HAVING - фільтрація груп після агрегації (аналог WHERE, але для агрегованих значень)
- FILTER - умовна агрегація - обчислення функції лише для рядків, що відповідають умові
- Віконна функція (Window Function) - функція, що виконує обчислення по групі рядків без їх згортання в один рядок
- OVER() - ключове слово, що визначає "вікно" (набір рядків) для віконної функції
- PARTITION BY - розбиття рядків на групи для віконної функції (аналог GROUP BY, але без згортання)

---

## Підзапити та CTE (Лекція 7)

- Підзапит (Subquery) - запит, вкладений всередину іншого SQL-запиту
- Скалярний підзапит (Scalar Subquery) - підзапит, що повертає одне значення (1 рядок, 1 колонка)
- Колонковий підзапит (Column Subquery) - підзапит, що повертає один стовпець із кількома рядками (використовується з IN, ANY, ALL)
- Табличний підзапит (Table Subquery) - підзапит, що повертає таблицю (використовується у FROM)
- Корельований підзапит (Correlated Subquery) - підзапит, що використовує значення з зовнішнього запиту, виконується для кожного рядка
- Некорельований підзапит (Non-Correlated Subquery) - підзапит, що не залежить від зовнішнього запиту, виконується один раз
- EXISTS - оператор, що повертає TRUE, якщо підзапит повернув хоча б один рядок
- CTE (Common Table Expression) - іменований тимчасовий набір даних, що існує протягом виконання запиту (WITH ... AS)
- Рекурсивний CTE (Recursive CTE) - CTE, що посилається сам на себе (WITH RECURSIVE) для обробки ієрархічних структур

## Нормалізація (Лекція 8)

- Нормалізація (Normalization) - процес організації таблиць для зменшення надмірності даних та усунення аномалій
- Нормальна форма (Normal Form, NF) - рівень організації таблиці відповідно до правил нормалізації
- Функціональна залежність (Functional Dependency, FD) - залежність X -> Y: кожному значенню X відповідає рівно одне значення Y
- Детермінант (Determinant) - ліва частина функціональної залежності (X у X -> Y)
- Повна функціональна залежність (Full FD) - атрибут залежить від усього складеного ключа, а не від його частини
- Часткова функціональна залежність (Partial FD) - атрибут залежить лише від частини складеного ключа
- Транзитивна залежність (Transitive Dependency) - ланцюжок залежностей A -> B -> C, де C непрямо залежить від A
- Багатозначна залежність (Multivalued Dependency) - X ->-> Y: для кожного X існує множина значень Y, незалежна від інших атрибутів
- Надключ (Superkey) - набір атрибутів, що однозначно ідентифікує запис
- Кандидат-ключ (Candidate Key) - мінімальний надключ (без зайвих атрибутів)
- 1НФ (First Normal Form, 1NF) - всі значення атомарні, немає повторюваних груп, кожен запис унікальний
- 2НФ (Second Normal Form, 2NF) - 1НФ + немає часткових залежностей від первинного ключа
- 3НФ (Third Normal Form, 3NF) - 2НФ + немає транзитивних залежностей
- НФБК (Boyce-Codd Normal Form, BCNF) - 3НФ + детермінант кожної залежності є надключем
- 4НФ (Fourth Normal Form, 4NF) - НФБК + немає багатозначних залежностей
- 5НФ (Fifth Normal Form, 5NF) - 4НФ + немає залежностей з'єднання, що не випливають з ключів
- Декомпозиція (Decomposition) - розбиття таблиці на кілька менших для досягнення вищої нормальної форми
- Аномалія вставки (Insert Anomaly) - неможливість додати дані без заповнення нерелевантних полів
- Аномалія оновлення (Update Anomaly) - необхідність оновлювати одні й ті ж дані в кількох місцях
- Аномалія видалення (Delete Anomaly) - втрата важливих даних при видаленні запису
- Надмірність даних (Data Redundancy) - зберігання одних і тих самих даних у кількох місцях

## Міграції (Лекція 9)

- Міграція (Migration) - версіонована послідовна зміна структури бази даних
- ALTER - DDL-команда для зміни існуючих структур (таблиць, колонок, індексів)
- Up-міграція (Up Migration) - міграція, що вносить зміни до структури БД
- Down-міграція (Down Migration) - парна міграція, що скасовує зміни up-міграції
- Таблиця версій (Schema Migrations Table) - таблиця, що зберігає інформацію про виконані міграції
- Чек-сума міграції (Migration Checksum) - хеш вмісту файлу міграції для контролю його незмінності після виконання
- Roll Forward - виправлення помилки через нову міграцію (рекомендований підхід)
- Rollback - скасування змін через down-міграцію
- Expand and Contract (Expand and Contract Pattern) - багатокрокова стратегія безпечної міграції без простоїв системи
- Безпечна ALTER операція (Safe ALTER) - операція, що виконується швидко і не блокує таблицю (ADD COLUMN, DROP COLUMN)
- Небезпечна ALTER операція (Unsafe ALTER) - операція, що може тривало блокувати таблицю (ADD NOT NULL, зміна типу)
- Ідемпотентність (Idempotency) - властивість, коли повторне виконання дає той самий результат, що й одноразове

---

## Транзакції та ACID (Лекція 10)

- Транзакція (Transaction) - набір запитів, що виконуються як одна неподільна операція: або всі, або жодна
- ACID (Atomicity, Consistency, Isolation, Durability) - набір властивостей транзакцій
- Атомарність (Atomicity) - всі операції транзакції виконуються повністю або не виконуються зовсім
- Консистентність (Consistency) - транзакція переводить БД з одного коректного стану в інший
- Ізоляція (Isolation) - паралельні транзакції не впливають одна на одну
- Персистентність (Durability) - після COMMIT зміни не можуть бути втрачені навіть при збої системи
- COMMIT - збереження всіх змін транзакції
- ROLLBACK - скасування всіх змін транзакції
- Рівень ізоляції (Isolation Level) - налаштування, що визначає ступінь ізоляції між паралельними транзакціями
- Read Uncommitted - найнижчий рівень ізоляції (в PostgreSQL працює як Read Committed)
- Read Committed - транзакція бачить лише закомічені дані (рівень за замовчуванням в PostgreSQL)
- Repeatable Read - значення прочитаних рядків не змінюються протягом транзакції
- Serializable - найвищий рівень ізоляції - повна серіалізація транзакцій
- Snapshot Isolation - кожна транзакція бачить свій "знімок" бази даних на момент початку
- Брудне читання (Dirty Read) - читання даних, які інша транзакція ще не закомітила
- Нерепетитивне читання (Non-Repeatable Read) - повторне читання того ж рядка повертає інше значення
- Фантомне читання (Phantom Read) - повторний запит повертає нові або зниклі рядки
- Втрачений апдейт (Lost Update) - дві транзакції читають і змінюють ті ж дані - одна перезаписує іншу
- Write Skew - дві транзакції змінюють різні рядки, але порушують спільне бізнес-правило
- Брудне писання (Dirty Write) - транзакція перезаписує незбережені зміни іншої транзакції
- SELECT FOR UPDATE - блокування вибраних рядків до кінця транзакції для запобігання конфліктам
- Deadlock - взаємне блокування двох транзакцій, що чекають одна на одну

## Індекси (Лекції 11-12)

- Індекс (Index) - допоміжна структура даних для пришвидшення пошуку, сортування та вибірки
- B-Tree - найпоширеніший тип індексу: збалансоване дерево з впорядкованими значеннями
- Hash-індекс (Hash Index) - індекс на основі хеш-таблиці, працює лише для перевірки рівності (=)
- Частковий індекс (Partial Index) - індекс лише для підмножини рядків, що відповідають умові (WHERE)
- Композитний індекс (Composite Index) - індекс по кількох колонках одночасно (порядок колонок важливий)
- Покриваючий індекс (Covering Index) - індекс, що містить всі колонки запиту (дозволяє Index Only Scan)
- Кардинальність (Cardinality) - кількість унікальних значень у колонці (висока - краще для індексу)
- Seq Scan (Sequential Scan) - повне послідовне сканування всіх рядків таблиці
- Index Scan - пошук за індексом + читання відповідних рядків з таблиці
- Index Only Scan - читання лише з індексу (всі потрібні дані вже в індексі)
- Bitmap Index Scan - створення бітової карти рядків з індексу для пакетного читання
- EXPLAIN - команда для перегляду плану виконання запиту
- EXPLAIN ANALYZE - EXPLAIN з фактичним виконанням запиту та вимірюванням часу
- Index Bloat - роздування індексу через накопичення мертвих версій кортежів
- Вузол дерева (Tree Node) - елемент B-Tree: кореневий, проміжний (branch) або листовий (leaf)
- Листовий вузол (Leaf Node) - найнижчий рівень B-Tree, містить значення ключів та посилання на рядки

## Денормалізація (Лекція 13)

- Денормалізація (Denormalization) - навмисне внесення надмірності до нормалізованих таблиць для підвищення швидкодії читання
- Похідна колонка (Derived Column) - додаткова колонка з дубльованими або обчисленими даними для уникнення JOIN
- Представлення (View) - збережений SQL-запит, що виконується при кожному зверненні (не зберігає дані)
- Матеріалізоване представлення (Materialized View) - фізично збережений результат запиту, що оновлюється явно (REFRESH MATERIALIZED VIEW)

---

## Фізична організація даних (Лекція 14)

- Кортеж (Tuple) - рядок таблиці у тому вигляді, як він зберігається на диску (заголовок + NULL bitmap + дані)
- Сторінка (Page) - одиниця зберігання даних на диску, фіксований розмір 8 КБ (за замовчуванням)
- Заголовок кортежу (Tuple Header) - метадані кортежу (23 байти): xmin, xmax, ctid, infomask та інші
- xmin - ID транзакції, що створила кортеж
- xmax - ID транзакції, що видалила кортеж (0 якщо не видалений)
- ctid (Current Tuple ID) - фізична адреса кортежу на сторінці (або посилання на нову версію)
- NULL Bitmap - набір бітів, що вказує які колонки містять NULL значення
- Heap File - неупорядкований набір сторінок, в яких зберігаються дані таблиці
- Покажчик на кортеж (Item Pointer) - запис у масиві на сторінці, що вказує на фізичне розташування кортежу
- MVCC (Multi-Version Concurrency Control) - механізм контролю паралельних транзакцій через зберігання кількох версій кортежів
- HOT Update (Heap-Only Tuple Update) - оптимізація: нова версія кортежу зберігається на тій самій сторінці
- TOAST (The Oversized-Attribute Storage Technique) - механізм зберігання великих значень (>2 КБ) в окремій спеціальній таблиці
- WAL (Write-Ahead Log) - журнал попереднього запису: всі зміни спочатку записуються в журнал, потім на диск
- LSN (Log Sequence Number) - унікальний ідентифікатор запису в WAL
- Checkpoint - момент запису всіх змінених сторінок з пам'яті на диск
- Dirty Page - сторінка, змінена в пам'яті, але ще не записана на диск
- Sequential I/O - послідовне читання/запис (швидке, оптимальне для дисків)
- Random I/O - читання/запис у різних місцях диску (повільне через переміщення голівки)
- VACUUM - процес очищення мертвих версій кортежів та повернення місця на сторінках

## Нереляційні СУБД (Лекція 15)

- NoSQL (Not Only SQL) - загальна назва для нереляційних СУБД
- CAP-теорема (CAP Theorem) - розподілена система може одночасно підтримувати лише 2 з 3 властивостей: Consistency, Availability, Partition Tolerance
- Консистентність (Consistency, CAP) - всі клієнти завжди бачать найновіші та однакові дані
- Доступність (Availability, CAP) - система доступна для читання та запису навіть при збоях
- Стійкість до розділення (Partition Tolerance, CAP) - система працює навіть при втраті зв'язку між серверами
- Вертикальне масштабування (Vertical Scaling) - збільшення потужності одного сервера (CPU, RAM, диск)
- Горизонтальне масштабування (Horizontal Scaling) - збільшення кількості серверів
- Реплікація (Replication) - копіювання даних на додаткові сервери для масштабування читання
- Шардинг (Sharding) - розподіл даних між кількома серверами для масштабування запису
- Документна СУБД (Document Database) - зберігає дані як JSON-документи без фіксованої схеми (MongoDB, CouchDB)
- Ключ-Значення СУБД (Key-Value Store) - зберігає пари ключ-значення, пошук лише за ключем, O(1) (Redis, Memcached)
- Column-Family СУБД (Column-Family Database) - зберігає набори колонок, масштабується лінійно, пошук за ключем (Cassandra, HBase)
- Графова СУБД (Graph Database) - зберігає дані як вершини та зв'язки, ефективна для графових запитів (Neo4j)
- Пошуковий рушій (Search Engine) - СУБД для повнотекстового пошуку на основі токенізації (ElasticSearch, OpenSearch)
- Колонкова СУБД (Columnar Database) - зберігає кожну колонку окремо, ефективна для OLAP запитів (Snowflake, BigQuery)
- Векторна СУБД (Vector Database) - зберігає числові вектори (embeddings) для семантичного пошуку (Pinecone, Qdrant, pgvector)
- Embedding - числовий вектор, що описує семантичне значення об'єкта (тексту, зображення)
- RAG (Retrieval-Augmented Generation) - паттерн, де LLM отримує контекст з векторної бази перед генерацією відповіді
- TTL (Time-To-Live) - час життя даних у кеші, після якого вони автоматично видаляються
- Polyglot Persistence - підхід з використанням кількох різних СУБД в одній системі
