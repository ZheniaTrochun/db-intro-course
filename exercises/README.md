# Вправи з SQL

## Навіщо потрібні вправи?

SQL - це практична навичка. Лекції дозволяють зрозуміти концепції, але справжнє розуміння приходить лише через практику.

Також, як приємний бонус, виконання вправ дозволяє заробити додаткові бали.  
Вірне рішення кожної вправи == 1 додатковий бал, тож таким чином можна набрати стільки додаткових балів, скільки Ви вирішите вправ.  
Можливість набрати повні додаткові бали актуальна **до початку залікового тижня** (опісля - бали лише в частковому розмірі).

## Структура вправ

Кожна вправа - це `.sql` файл з описом завдання у коментарях та місцем для вашого рішення після рядка `-- Рішення:`.

### GROUP BY та віконні функції (директорія `group_by/`)

| Файл                        | Завдання                                                                             |
|-----------------------------|--------------------------------------------------------------------------------------|
| `enrolment_stats.sql`       | Порахувати статистику записів на курси для кожного року навчання                     |
| `popular_courses.sql`       | Знайти курси, на які записано більше ніж 100 студентів                               |
| `student_average_grade.sql` | Для кожного студента знайти його середній бал у порівнянні з середнім балом по групі |
| `student_success_rate.sql`  | Порахувати успішність студентів залежно від року навчання                            |
| `successful_groups.sql`     | Знайти групи, в яких середній бал студентів вищий за 75                              |

### Підзапити (директорія `subqueries/`)

| Файл                                    | Завдання                                                                |
|-----------------------------------------|-------------------------------------------------------------------------|
| `above_average_students_per_group.sql`  | Знайти студентів, чий середній бал перевищує середній бал їхньої групи  |
| `busiest_teachers.sql`                  | Знайти топ-100 викладачів, що мають найбільшу кількість кредитів        |
| `min_semester_for_course.sql`           | Для кожного курсу знайти мінімальний семестр, в якому він може читатись |
| `students_with_many_enrolments.sql`     | Знайти всіх студентів, які записані на більше курсів ніж в середньому   |
| `top_5_students.sql`                    | Знайти топ-5 студентів у кожному курсі за отриманими балами             |

## Як виконувати вправи

1. Відкрити `.sql` файл вправи, прочитати умову завдання в коментарях - там вказано очікувані колонки та сортування
2. Написати SQL-запит, що вирішує завдання після рядка `-- Рішення:`
3. Підчас написання запиту, перевіряйте його на тестових даних
4. Запустіть тести для перевірки

### Запуск тестів

З директорії `exercises/` виконайте:

```bash
./run_all_tests.sh
```

Скрипт автоматично створить віртуальне середовище Python, встановить залежності та запустить тести на трьох наборах даних (`base`, `10k`, `100k`).  
Результати будуть збережені у `tests/test_results/`.

Для запуску тестів необхідно:
- запущений Docker
- Python 3.13

**Якщо у Вас ОС Windows - команду потрібно запускати у Git Bash або у WSL**

## Як здати рішення та отримати бали

1. Створіть форк цього репозиторію на GitHub
2. Реалізуйте розв'язання задач у відповідних `.sql` файлах
3. Перевірте рішення локально:
   ```bash
   cd exercises
   ./run_all_tests.sh
   ```
4. Закомітьте та запуште зміни
5. Відкрийте Pull Request на GitHub:
   - Перейдіть на сторінку вашого форку
   - Натисніть "Compare & pull request"
   - У назві PR вкажіть: `Вправи - <Прізвище> <Група>`
   - Заповніть опис PR
   - Натисніть "Create pull request"
6. Дочекайтесь результатів автоматичних тестів у PR
